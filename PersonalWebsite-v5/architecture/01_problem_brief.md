# 01 Problem Brief: Gideon Valera Portfolio

## 1. Context
What it's for: A personal portfolio website for personal projects, detailing necessary scripts, and highlighting a transition into ADLC (AI Development Life Cycle) for Gideon Noel S. Valera.
Type of business: Tech Professional Portfolio
Main button text: "Connect with me"

## 2. The Core Challenge
Building a single-page React application that doesn't feel like a generic developer site. It must bridge the gap between "Hardcore API Dev" and "Friendly Project Leader." Pick a color palette that feels right for this type of business. Don't overthink it — just choose colors that look great together and fit the vibe.

## 3. Scope of Work
- **Visual Design:** Tailwind implementation of a warm, wellness-inspired color palette. Look for the branding guidelines similar to this: https://solo-sdlc.razcarcoana.com/
- **Content:** 6 distinct sections (Hero, How it Works, Features, Trusted By, Bottom CTA, Footer). Page sections (build them in this order):

Hero — Big bold headline (8 words max, e.g., "Architecting Solutions & Mastering the ADLC"), one-line description underneath mentioning I am a results-driven Technical Project Manager bridging complex business requirements and technical execution, and a prominent call-to-action button. Add an AI-generated, professional yet quirky/approachable image or illustration on the right side.
+2

How it works — 3 simple numbered steps showing my project lifecycle.

Step 1: Strategy & Scoping (Digital roadmap scoping and stakeholder management).
+1

Step 2: Scripts & Architecture (RESTful API integrations, JSON/XML payloads, and VBA automation).
+2

Step 3: ADLC & Deployment. Connect them visually.

Features — A grid of 4 benefit cards focusing on my core offerings.

Card 1: eCommerce Ecosystems and Systems Design (expertise in Magento, Drupal, WordPress).

Card 2: Full-Stack Data Flow (optimizing data retrieval and endpoints).

Card 3: Custom Scripts & Automation thru AI Development Life Cycle. Focus on what the customer gets (efficiency, scalable architecture), not just the technical features.

Card 4:  Technology Project Management, Business Analysis and Consultancy

Works with — A row of recognizable text/logos representing past experience with a "Trusted by" label. Include AXA Philippines, Globe Group, Cosme de net, AktivAsia, LeoPC, Cup n' Grind, The Pack Labs, and Johnson & Johnson. Keep it compact and credible.
+2

Bottom CTA — A full-width section near the end with a bold headline (e.g., "Ready to build something brilliantly efficient?") and the main call-to-action button. Last push before the footer.

Footer — Logo, navigation links in columns, social media icons, and copyright text. Include my contact details: gideon.valera@gmail.com and Caloocan City, Metro Manila. Clean and organized.
+1

- **Tech Requirements:** React, Tailwind, Responsive Design, Animations. Use real, realistic placeholder text — not lorem ipsum. Write copy that actually fits this business. The tone must be professional but quirky, approachable, and highly competent. Add smooth, subtle animations (fade in on scroll, hover effects on buttons and cards). The call-to-action button should appear above the fold and repeat near the bottom. Keep the code clean and production-ready. Single page, no routing
- **Payload:** A deployment-ready `index.html` (bundled via build process or single source).

## 4. Constraints
- Must use rounded UI elements.
- Must repeat CTA above fold and at bottom.
- Must use professional-quirky copy.
- No external routing libraries (Single Page).
